General Knowledge

Tree

balanced search trees

  • guaranteed height of O(log n) for n items

red-black tree

basic

  • some specific type of red-black tree
  • A node is either red or black
  • The root and leaves (NIL) are black
  • If a node is red, then its children are black
  • All paths from a node to its NIL descendants contain the same number of black nodes

black height of the red-black tree = 2

  • each node has its own black height
  • Nodes requite one storage bit to keep track of color
  • The longest path (root to farthest NIL) is no more than twice the length of the shortest path (root to nearest NIL)
    • Shortest path: all black nodes
    • Longest path: alternating red and black
  • Time conplexity:
    • Search O(log n)
    • Insert O(log n)
    • Remove O(log n)
  • Space complexity: O(n)

operatoin

rotation: O(1)

  • alter the structure of a tree by rearranging subtrees
  • goal is to decrease the height of the tree
    • Red-black trees: maximum height of O(log n)
    • Larget subtrees up, smaller subtrees down
  • Does not affect the order of elements

insertion

  • Strategy:
    • Insert Z and color it red
    • Recolor and rotate nodes to fix violation

  • 4 cases:

    • Case 0: Z = root
      • Solution: color z black
    • Case 1: Z.uncle = red
      • recolor z’s parent/grandparent and uncle
    • Case 2: z.uncle = black (triangle)
      • rotate z.parent (as in z takes place of z,parent)
    • Case 3: x.uncle = black (line)
      • Rotate z’grandparent in the opposite direction of z
      • and recolor the orginal parent and grandparent of z after the rotation

  • psudo code

  • Time complexity
    • Insert: O(log n) <- maximum height of red-black trees
    • Color red: O(1)
    • Fix violations: O(logn)
      • Constant # of:
        • Recolor: O(1)
        • Rotation: O(1)
    • Total: O(log n)

HashMap

HashMap Node

Hashmap is a k-v pair set, each node is represented by Node

Each node also includes the reference to the next node.

HashMap data structure

HashMap data structure is array + (linked list or red-black tree)

Step1: calculate: hashcode of “key1”, then hash the hashcode to get the position of the array

Hashcode

If an object is not modified, then it’s hashcode will always be the same.

If two objects are not the same, then their hashcode also might be the same.

Storage

  • First stage
    • Each node has a next attribute which points to the next node.
    • Array + linkedlist
  • Second stage
    • too many elements in linkedlist would affect the search efficiency.
    • So when the #elements in the linkedlist reach 8 and the length of array reaches 64, we use red-black trees instead of linkedlist for better search efficiency.

Two important parameters

  • Initial capcity
    • the length of the array, default value is 16
  • Load factor
    • val = Load factor * capacity. Once the #elements in the array exceed “val”, use “refresh” to enlarge the array so that we can have a new array with larger capacity.
    • Notice: all the elements in the old array to be hashed and reallocated to the new array. SO it’s a very expensive operation.
