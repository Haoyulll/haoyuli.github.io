Leetcode 509. Fibonacci Number

https://leetcode.com/problems/fibonacci-number/

solution 1: naive recursive

  • Time complexity = #childproblem * the time of each childproblem= O(2^n)

Suppose that the target is f(20), then we need to know f(19) and f(18). As for f(19), we need to get f(18) and f(17) … until we reach f(1) and f(2) for each branch. We can use binary tree to demonstate the process.

-#childproblem = #binaryTree node = O(2^n) (n is the height of the binary tree)

the time of each childproblem, O(1), f(n-1) +f(n-2)

  • Problem: repeat calculations 重叠子问题

For example, f(18) will be calcuted twice, one fot f(18), the other one for f(19)

solution 2: recursive with notebook

Reduce repeat calculation by recoding the calculated childnode value

  • time complexity O(n)

solution 3: bottom to top (dp 数组)

dp数组的定义：当target为i时，result 为dp[i]

math model 状态转移方程

Leetcode 322. coin change

https://leetcode.com/problems/coin-change/

Return the fewest number of coins that you need to make up that amount. If that amount of money cannot be made up by any combination of the coins, return -1

solution1:recursive + loop

Base case: amount. = 0, return 0

math model 状态转移方程

递归树

problem: 重叠子问题

time complexity

Suppose target amount is n, coins number is k. In the worst case, the height is n the fanout is k. So number of child node is k^n.

For each childProblem, there is a for loop. O(k).

So the time complexity overall is O(k^n)

solution2:recursive + loop with memo

time complexity: O(kv)

子问题总数：O(n)

处理一个子问题的时间：O(k)

solution3: bottom to top

time complexity: O(kv)

KMP字符匹配算法

Leetcode 28 Strstr()

https://leetcode.com/problems/find-the-index-of-the-first-occurrence-in-a-string/

找到子str在父str中的起始位置，otherwise return -1

solution1: naive

time complexity

O(MN) - M the length of haystack, N is the length of needle

space complexity

O(1)

Problem

  • double for loop -> slow
  • if there are many repeat letter, then

solution2: KMP 算法

KMP 算法永不回退haystack的指针 i，不走回头路（不会重复扫描 haystack), 而是借助dp数组中储存的信息把pat移动到正确的位置继续匹配。 由于不走回头路，所以时间复杂度只需要O(N), 用空间换时间。
